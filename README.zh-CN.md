# Image Collection v2

[English](README.md) | [繁體中文](README.zh-TW.md) | 简体中文

通过 IMEI 号码从 NAS 共享文件夹批量收集设备图像的桌面工具。使用 Electron、React 及 Liquid Glass UI 主题开发。**v1.5.1 — MR 可靠性与诊断。**

操作人员导入审计清单 — 建议包含 IMEI、机器及日期列以获得最快结果 — 选择要搜索的机器文件夹，并以可配置的整理方式将匹配的图像文件夹导出至本地目标位置。

---

## 快速开始

1. **安装** — 运行 `Image Collection v2 Setup X.X.X.exe` 并按照提示操作。
2. **设置来源** — 点击共享文件夹旁的**浏览**，选择 NAS 根目录（例如 `Z:\`）。机器文件夹将自动填充。
3. **导入审计清单** — 在审计面板点击**浏览**，或拖放 CSV / XLSX 文件。**为获得最佳结果，请包含 IMEI、机器及日期列** — Smart Search 会自动检测这些列、自动选中相关机器文件夹、自动填充日期范围，并将搜索时间从数分钟缩短至数秒。
4. **验证检测结果** — 检查审计面板中的绿色标记，确认机器及日期列已被检测。确认来源面板中仅选中了相关的机器文件夹。若检测失败或数据不准确，请关闭 Smart Search 以回退到仅限 IMEI 模式。
5. **配置设置** — 选择动作、图像类型、整理模式，并设置目标位置。
6. **搜索** — 点击**开始搜索**。匹配结果将实时流式显示在结果表格中。
7. **导出** — 查看结果后，点击**导出结果**。进度显示在状态栏中。

## 功能

| 功能 | 说明 |
|---|---|
| **多格式审计导入** | CSV、XLSX、XLS、TXT — 自动检测 IMEI 列并跳过标题行 |
| **Smart Search** | 自动检测审计文件中的机器及日期列，进行定向文件夹查找，取代完整 NAS 扫描；自动选中机器文件夹并填充日期范围 |

| **MR PASS / FAIL 收集** | 通过切换开关从 `ModelRecogImages/` 收集模型识别图像 |
| **仅 AI 图像** | 仅从匹配的 IMEI 文件夹导出 `FD/` 子文件夹（AI 检测图像） |

| **缺失 IMEI 报告** | 查看并保存在 NAS 上未找到的审计 IMEI 清单 |
| **深色 / 浅色主题** | 两种模式均采用 Liquid Glass 主题；跨会话保存 |
| **三语** | 英文、繁体中文及简体中文 — 通过来源面板的语言按钮切换 |
| **设置持久化** | 所有偏好设置通过 electron-store 保存，并在下次启动时恢复 |

## 来源配置

### 共享文件夹

输入或浏览至 NAS 挂载点（例如 `Z:\`、`\\NAS_Lonestar\share`）。工具会扫描第一层子目录并填充文件夹切换网格。

> **不限于 NAS。** PC 可访问的任何驱动器均可搜索 — 包括连接至 SMART Grade PC 或 RPC 的本地硬盘 — 只要文件夹结构符合标准的 `机器 → YYYYMMDD → IMEI_Index` 层级即可。

- **机器文件夹**（名称匹配 `M` + 数字，如 `M8`、`M10`）默认为选中。
- **实用文件夹**（`audits`、`crackimages`、`GRR Images`、`version_control`）默认为不选中。
- `#recycle` 及 `$RECYCLE.BIN` 始终隐藏。

使用**全选 / 取消全选**进行批量切换。**刷新**可重新扫描来源以更新新增或移除的子文件夹。

### 多来源管理

将常用的 NAS 路径保存为命名来源：

1. 浏览至 NAS 根目录。
2. 点击 **[+]** 并输入名称（例如"NAS Lonestar"）。
3. 按下 Enter 保存。

通过下拉菜单切换来源。每个来源各自记忆其文件夹切换状态。点击 **[-]** 可移除来源（因为此操作无法撤消，会显示确认提示）。

首次启动时，先前保存的单一路径设置会自动迁移为命名来源。

### 日期范围

将搜索限制在特定日期区间的文件夹：

- **起始 / 结束日期** — 仅扫描 `YYYYMMDD` 名称落在此范围内的文件夹。两者均留空则搜索所有日期。
- 若起始日期晚于结束日期，会显示警告（不会有任何匹配结果）。

## 审计清单导入

拖放或浏览以导入审计文件。

### 推荐方式：Smart Search（IMEI + 机器 + 日期）

收集图像的最快方式是在审计文件（CSV 或 Excel）中包含 **IMEI**、**机器**及**日期**列。当三列均存在时，Smart Search 会直接前往 NAS 上每个 IMEI 的确切文件夹 — 将搜索时间从数分钟缩短至数秒。

**推荐的审计文件格式：**

| IMEI | 机器 | 日期 |
|---|---|---|
| 350308317018557 | M14 | 2026-05-14 |
| 350448886316469 | M21 | 2026-05-14 |
| 351101754222413 | M22 | 2026-05-14 |

列标题可以是任意文字 — 解析器通过数据模式识别列，而非依据标题名称。列顺序无关。额外列会被忽略。IMEI 列始终通过 15 位数字值来识别。

**机器列** — 可识别以下名称格式：

| 文件中的值 | 标准化为 |
|---|---|
| M8, M10 | M8, M10（完全匹配） |
| M08, M-8, M-08 | M8（去除前导零 / 连字符） |
| SG-M16, LAX-M08 | M16, M8（去除站点前缀） |
| Machine 8 | M8（提取数字） |
| 08, 8 | M8（纯数字） |

**日期列** — 可识别以下日期格式：

| 格式 | 示例 |
|---|---|
| YYYYMMDD | 20260515 |
| YYYY-MM-DD | 2026-05-15 |
| YYYY/MM/DD | 2026/05/15 |
| MM/DD/YYYY | 05/15/2026 |
| MM-DD-YYYY | 05-15-2026 |
| M/D/YYYY | 5/15/2026 |
| M/D/YY | 5/15/26 |
| M/D/YY HH:MM | 5/14/26 11:57（自动去除时间部分） |

任何尾随的时间部分（例如 `11:57`、`11:57:00` 或 ISO `T` 分隔符）在解析前会被去除。对于模糊的日期格式（MM/DD 与 DD/MM），解析器会采样所有数据行以判断正确的解读方式。完全模糊时默认使用 MM/DD（美国格式）。

**导入后**，审计面板以颜色标记显示检测质量：
- **绿色**：所有数据行解析成功
- **橙色**：80% 以上数据行解析成功（部分值无法识别）
- **红色**：低于 80% 解析成功（列检测可能不可靠）

**自动填充日期范围**：当检测到日期列时，来源面板的起始 / 结束日期字段会自动填入审计数据中最早及最晚的日期。这可缩小搜索范围，并提供日期解析正确的视觉确认。如有需要，可手动调整日期。

**自动选择机器文件夹**：当检测到机器列时，来源面板会自动选中审计数据中引用的机器文件夹，并取消选中其余文件夹。例如，若审计文件仅包含 M14、M21 和 M22 的条目，则仅这三个文件夹会被选中。这可避免扫描无关的机器，加快广泛扫描的回退速度。若未检测到机器列，所有机器文件夹将维持先前的切换状态。

**部分提示**：审计文件不需要包含全部三列。Smart Search 会根据可用的列进行调整：

| 可用列 | 搜索行为 |
|---|---|
| IMEI + 机器 + 日期 | 直接文件夹查找 — 最快（数秒） |
| 仅 IMEI + 机器 | 广泛扫描但限缩至每个 IMEI 提示的机器 |
| 仅 IMEI | 跨所有已选机器进行完整广泛扫描（最慢） |

缺失值或值无法识别的 IMEI 会自动回退到下一层广泛扫描 — 不会丢失任何结果，这些 IMEI 只是搜索较慢。

**MR PASS / FAIL 搜索**同样受益于 Smart Search。MR 收集执行的就是标准 IMEI 文件夹搜索，因此当提示可用时，它会和普通搜索一样直接前往 `Machine/Date/{IMEI}_{index}/`，再从中取出 `SG-*.png`，而非先发现所有日期文件夹。

若检测到的列不准确，请关闭 **Smart Search** 以回退到仅限 IMEI 模式。

### 备选方案：仅限 IMEI 导入

若您的审计数据仅包含 IMEI（无机器或日期列），工具会回退为广泛 NAS 扫描 — 搜索所有已选机器中的每个日期文件夹。此方式可行但速度明显较慢。

| 格式 | 行为 |
|---|---|
| **CSV** | 通过采样前 20 行自动检测 IMEI 列。跳过标题行。支持逗号、Tab 及分号分隔符。 |
| **XLSX / XLS** | 读取第一个工作表。自动检测 IMEI 列。跳过标题行。 |
| **TXT** | 每行一个值。每行视为潜在的 IMEI。TXT 文件无法使用 Smart Search（单列）。 |

**IMEI 验证**：必须为恰好 15 位数字。无效条目及重复条目会被计数并报告。

> **OneDrive 文件：** 若您的审计文件位于设置为"仅在线"的 OneDrive 文件夹中，首次读取可能会在 OneDrive 下载文件时卡住。本工具现在会在 20 秒后超时，并显示清晰的消息而非无限挂起。若要彻底避免此问题，请在文件资源管理器中右键点击该文件 →**始终保留在此设备上**（等待绿色勾号出现），或将审计文件保存在普通的本地文件夹中。

## 设置参考

### 动作

| 值 | 行为 |
|---|---|
| **复制**（默认） | 将匹配的文件夹复制到目标位置。来源数据不受影响。 |
| **移动** | 传输文件夹并在复制后**删除来源**。启用前会显示确认对话框。下次启动时，动作始终重置为复制以确保安全。 |

> 移动模式**不适用于** MR 图像导出 — MR 图像始终以复制方式处理，以保护共享的 `ModelRecogImages/` 目录。

### 图像类型

| 值 | 行为 |
|---|---|
| **全部**（默认） | 导出匹配文件夹中的所有图像文件。 |
| **BMP** | 仅导出 `.bmp` 文件。 |
| **JPEG** | 仅导出 `.jpg` / `.jpeg` 文件。 |

### 整理

控制导出文件夹在目标位置的排列方式：

| 模式 | 结构 | 示例 |
|---|---|---|
| **平铺** | 全部放在同一文件夹 | `dest/350002267153742_192/` |
| **按机器** | 按来源机器分组 | `dest/M8/350002267153742_192/` |
| **按日期** | 按扫描日期分组 | `dest/20260515/350002267153742_192/` |
| **机器 > 日期** | 两层：先机器后日期 | `dest/M8/20260515/350002267153742_192/` |
| **日期 > 机器** | 两层：先日期后机器 | `dest/20260515/M8/350002267153742_192/` |
| **按 IMEI** | 将同一设备的所有实例分组 | `dest/350002267153742/M8_20260515_192/` |
| **按型号** | 按设备型号分组（从 MR 图像解析） | `dest/Apple-iPhone13Pro/350002267153742_192/` |
| **机器 > 型号** | 两层：先机器后设备型号 | `dest/M8/Apple-iPhone13Pro/350002267153742_192/` |

**MR 图像导出**时，每个 IMEI 会有自己的文件夹，包含匹配的 `.png` 文件：

| 模式 | MR 示例 |
|---|---|
| 平铺 | `dest/350002267153742/M8_20260515_Samsung-Galaxy_S24.png` |
| 机器 > 日期 | `dest/M8/20260515/350002267153742/Samsung-Galaxy_S24.png` |
| 按 IMEI | `dest/350002267153742/M8_20260515_Samsung-Galaxy_S24.png` |
| 按型号 | `dest/Samsung-Galaxy_S24/350002267153742/M8_20260515_Samsung-Galaxy_S24.png` |
| 机器 > 型号 | `dest/M8/Samsung-Galaxy_S24/350002267153742/20260515_Samsung-Galaxy_S24.png` |

> **始终包含：** 每次导出都会自动包含每个 IMEI 文件夹中的 `DefectLog.xml` 和 MR 图像（`SG-*.png`），不受图像类型筛选设置影响。这些文件对审计和型号识别至关重要。

> **提示：** 整理模式独立于搜索 — 您可以在搜索完成后、导出前更改模式。导出将使用您点击**导出结果**时所选的模式。

### 重复

| 值 | 行为 |
|---|---|
| **跳过**（默认） | 若目标文件夹 / 文件已存在，不进行更改。 |
| **覆盖** | 以新的来源数据替换现有的目标文件夹 / 文件。 |

### 扫描索引

| 值 | 行为 |
|---|---|
| **全部**（默认） | 包含每个扫描索引（`_1`、`_2`、`_3` 等）。 |
| **仅第一个** | 仅 `_1` 条目 — 该设备在该机器 / 日期的系列扫描中的第一个。 |

### MR PASS / MR FAIL — 模型识别图像收集

这两个切换开关会为审计清单中的每个 IMEI 收集该设备的**模型识别（MR）图像** — 也就是系统捕获的那一张 `SG-*.png`。当您的审计文件是一份由评级系统标记的设备清单（例如"颜色错误"报告），而您只需要每个设备的 MR 捕获图像时，这是正确的选择。

**开启 MR PASS 或 MR FAIL 任一个即会启用 MR 收集。** 它会执行与普通搜索相同的快速 IMEI 文件夹查找，然后只从每个设备的文件夹中取出 `SG-*.png`：

```
Machine/{date}/{IMEI}_{index}/SG-{machine}-{code}-{IMEI}-{brand}-{model}.png
```

MR 图像就位于每个设备的 IMEI 文件夹内，与扫描图像并列 — 因此收集**很快**（文件夹小，直接按 IMEI 查找）。本工具**不会**枚举 `ModelRecogImages/{Brand-Model}/` 文件夹 — 这些文件夹累积了数万个文件，数量过于庞大，无法通过网络扫描。

- 结果会**按图像的型号名称标记 PASS/FAIL**（`Error-Error` → FAIL／红色，其他任何值 → PASS／绿色）。
- 型号从文件名解析（例如 `Apple-iPhone8`），并显示在结果表格中。
- 启用时，**MR 收集会取代标准图像收集** — 只会导出 `SG-*.png`，而非整个文件夹。
- 若某设备的文件夹有找到但其中不含 `SG-*.png`，则会报告为缺失（并计入搜索日志）。

> 审计清单本身就是筛选条件 — 没有"颜色错误"切换开关，因为搜索无法检测评级。加载清单、启用 MR，即可收集每个列出设备的 MR 图像。

### 仅 AI 图像

启用时，仅从每个匹配的 IMEI 文件夹导出 `FD/` 子文件夹内容（AI 检测图像）。文件夹根目录的标准扫描图像会被排除。

禁用时，会导出完整的 IMEI 文件夹，包含 `FD/` 作为子文件夹。

### 目标位置

导出图像写入的本地文件夹。浏览或输入路径。跨会话保存。

## 结果

搜索完成后，结果面板显示：

- **摘要栏** — 找到的 IMEI 总数 vs. 审计清单中的总数、耗时。
- **状态圆点** — 绿色（完成）、橙色（不完整：文件数量明显低于中位数）、红色（缺失：NAS 上未找到）。
- **可排序表格** — 点击任意列标题（IMEI、机器、日期、索引、文件）进行升序 / 降序排序。
- **文件明细** — 每个匹配项显示总文件数，并以 `(Xb Yj)` 格式显示 BMP 及 JPEG 数量。
- **查看缺失的 IMEI** — 切换表格以显示未匹配的审计条目。
- **保存缺失的 IMEI** — 将缺失清单导出为 `.txt` 或 `.csv` 文件。

### 不完整检测

当匹配项的文件数量低于所有匹配项文件数量中位数的 50% 时，会被标记为不完整（橙色圆点）。这可以发现存在但图像数量异常少的 IMEI 文件夹。

### 错误反馈

如果搜索或导出遇到错误（网络中断、目标位置无法访问、权限问题），会出现错误横幅显示具体信息。横幅在下次操作开始时自动清除，也可手动关闭。

如果搜索期间有文件夹无法访问，状态栏会显示计数（例如「3 个访问错误」）。这可以区分真正缺失的 IMEI 与因网络或权限问题而无法检查的 IMEI。

## 日志

每次搜索及导出都会在应用程序的用户数据目录中生成详细的诊断日志文件：

```
%APPDATA%/Image Collection v2/logs/search-YYYYMMDD-HHmmss-mmm.log
%APPDATA%/Image Collection v2/logs/export-YYYYMMDD-HHmmss-mmm.log
```

**搜索日志**准确记录搜索的执行内容 — 当搜索返回的结果比预期少时极为实用：
- **标题** — 模式（标准 vs MR）、IMEI 数量、提示数量、Smart Search 开 / 关、MR PASS/FAIL、扫描索引筛选、日期范围、已选文件夹、NAS 根目录。
- **路径决策** — 搜索是以定向方式（直接文件夹查找）或完整发现执行、构建的日期文件夹数量、每机器计数，以及有多少 IMEI 落入各层回退（及其原因）。
- **摘要** — 找到的匹配数、缺失数、已扫描文件夹数、扫描错误数、耗时。

**导出日志**记录逐文件夹的来源→目标路径、每个已复制文件及其大小、吞吐量，以及跳过 / 失败数量。

工具会保留每种类型**最近 3 份**日志文件，并自动轮换较旧的文件（搜索与导出各自独立轮换）。

点击状态栏中的**查看日志**会打开日志文件夹。

## 搜索历史

最近 5 次已完成的搜索会被存储，可通过操作栏中的**历史**按钮访问。每个条目显示：

- 搜索的日期和时间
- 来源名称及 NAS 路径
- 审计文件名称及 IMEI 数量
- 匹配数量及缺失数量
- 搜索耗时
- 日期范围（若有筛选）
- MR 标记（若 MR 切换为启用）

## NAS 目录结构

工具预期 NAS 上具有以下层级结构：

```
NAS_ROOT/                              （例如 Z:\）
  M8/                                  机器文件夹（第 1 层）
    20260515/                           日期文件夹（第 2 层，YYYYMMDD）
      350002267153742_192/              IMEI_索引文件夹（第 3 层）
        image1.bmp                      图像文件（第 4 层）
        image2.jpg
        FD/                             AI 检测图像子文件夹
          fd_image1.bmp
      350024510270586_85/
    20260514/
    ModelRecogImages/                   MR 图像库（本工具不扫描 — 过于庞大；MR 改用各 IMEI 文件夹内的 SG-*.png）
      20260515/
        Apple-iPhone8/                  Brand-Model 子文件夹（MR PASS）
          SG-M008-075545-358627090247469-Apple-iPhone8.png
        Error-Error/                    错误子文件夹（MR FAIL）
          SG-M008-074837-359814715825890-Apple-iPhoneXR.png
    Bin/                                搜索时跳过
  M10/
  ...
  #recycle/                             始终隐藏
  audits/                               实用文件夹（默认不选中）
```

**命名约定：**
- IMEI：始终为 15 位数字。
- 扫描索引：下划线后的整数（`_1`、`_2`、`_3` 等）— 该设备在系列扫描中的顺序编号。
- 日期文件夹：始终为 8 位数字，`YYYYMMDD` 格式。
- MR 图像文件名：`SG-{machine}-{code}-{IMEI}-{brand}-{model}.png`（IMEI 为第 4 个连字符分隔的段）。

## 版本历史

### v1.5.1 — MR 可靠性与诊断（2026-06-10）

源于一次真实的"颜色错误"收集，397 张图像中返回了 0 张。已找出并修复根本原因：

**MR 收集 — 现在快速且可靠：**
- MR 收集（启用 MR PASS 或 MR FAIL **任一个**）现在使用**快速 IMEI 文件夹查找**，只从每个设备的文件夹中取出那张 `SG-*.png` MR 图像。MR 图像就位于每个 IMEI 文件夹内，与扫描图像并列，因此收集可在数秒内完成。
- 它不再枚举 `ModelRecogImages/{Brand-Model}/` 文件夹 — 这些文件夹累积了数万个文件，通过网络扫描会超时（这正是先前返回空白结果与 UI 卡住的根本原因）。
- 结果会按图像的型号名称标记为 PASS（绿色）/ FAIL（红色）；只会导出 `SG-*.png`。
- "颜色错误"只能从审计清单得知，永远无法从 NAS 得知 — 因此清单就是筛选条件；没有"颜色错误"切换开关。

**日期处理：**
- 自动填充的**结束日期现在是最后测试日期 + 1 天**，因此在接近午夜时测试的设备（文件夹换到隔天）不再被遗漏。
- 缺失 / 不存在的文件夹（ENOENT）被视为良性，不再计为扫描错误。

**诊断：**
- **搜索现在会被记录**（`search-*.log`），包含完整请求、定向 vs 发现的路径决策、每机器文件夹计数、回退转换，以及结果摘要，让短促的搜索也能一目了然地诊断。**查看日志**会打开日志文件夹。

**可靠性：**
- **过期状态修复** — 日期范围会在搜索时从已提交状态读取，且机器自动选择以内容为键。更改设置 / 切换开关后，不再需要重新上传审计文件搜索才能运作。
- **OneDrive 安全读取** — 20 秒超时并显示清晰的"仍在下载"消息，而非无限挂起。

**新增：**
- **机器 → 型号**整理模式（`dest/M8/Apple-iPhone13Pro/IMEI_index/`）。MR 匹配现在会带有解析出的型号名称。

### v1.4.0 — 强化版本（2026-05-28）

全面可靠性与安全性审计。识别并修复 30 个问题，涵盖所有严重程度等级。

**关键修复：**
- **移动模式安全性** — 当图像类型筛选启用时（会破坏未选取的文件）、未复制任何文件时、或复制失败时，阻止源删除
- **原子覆写** — 覆写前备份目标文件夹，失败时自动还原，防止中断导出导致数据丢失
- **日志防崩溃保护** — 流错误处理器防止未处理的异常导致应用程序崩溃

**可靠性改进：**
- 目标路径在源路径内的验证，防止递归复制循环
- 跳过现有文件夹的完整性检查，不足 50% 时重新导出
- 连续失败 10 次的断路器机制（网络中断检测）
- NAS readdir 超时（15 秒），防止无响应共享时无限挂起
- AI 图像模式的递归文件计数（计算 FD/ 子文件夹内容）
- MR 扫描错误现在被跟踪和报告，而非静默吞噬
- Smart Search 机器限定回退将未找到的 IMEI 送入广泛扫描

**导出改进：**
- Windows 长路径支持（路径 > 240 字符时使用 \\\\?\\ 前缀）
- 机器/文件夹/型号名称的路径遍历净化
- 导出开始前的目标可写入性测试
- 移动模式源删除失败现在准确报告在结果中
- 导出完成后的"打开文件夹"按钮

**搜索改进：**
- 扫描错误详情显示在结果中（路径 + 错误码）
- 扫描索引筛选计数的跟踪和报告
- MR IMEI 提取扫描所有文件名段（更健壮的格式变体处理）
- 型号名称净化防止导出路径中的路径遍历
- 文件计数的有界并行（每个工作者最多 8 个）

**UI/UX 改进：**
- 首次启动时以工作流程指引取代空白的"就绪"状态
- 常见失败的用户友好错误消息（ENOENT、EPERM、ENOSPC、ETIMEDOUT）
- 加载新审计文件时清除过期结果
- 防止重复搜索的双击保护
- 流式匹配计数器使用 requestAnimationFrame 防抖
- 缺失 IMEI 列表分页（防止大列表时浏览器冻结）
- 审计文件拖放的扩展名验证
- 所有设置的改进工具提示，使用易懂的语言描述
- 不完整检测阈值已修正（原有差一错误）

**解析器改进：**
- 重复 IMEI 提示保留首次出现（而非最后）
- 纯数字机器模式限制为 1-99 范围
- 不支持的文件格式现在以明确错误拒绝

运行 NSIS 安装程序。它将：
