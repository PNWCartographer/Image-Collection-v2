# Image Collection v2

[English](README.md) | 繁體中文 | [简体中文](README.zh-CN.md)

透過 IMEI 號碼從 NAS 共享資料夾批次收集裝置影像的桌面工具。使用 Electron、React 及 Liquid Glass UI 主題開發。

操作人員匯入稽核清單 — 建議包含 IMEI、機器及日期欄位以獲得最快結果 — 選擇要搜尋的 NAS 機器資料夾，並以可設定的整理方式將匹配的影像資料夾匯出至本機目標位置。

---

## 快速開始

1. **安裝** — 執行 `Image Collection v2 Setup X.X.X.exe` 並依照提示操作。
2. **設定來源** — 點擊共享資料夾旁的**瀏覽**，選擇 NAS 根目錄（例如 `Z:\`）。機器資料夾將自動填入。
3. **匯入稽核清單** — 在稽核面板點擊**瀏覽**，或拖放 CSV / XLSX 檔案。**為獲得最佳結果，請包含 IMEI、機器及日期欄位** — Smart Search 會自動偵測這些欄位、自動勾選相關機器資料夾、自動填入日期範圍，並將搜尋時間從數分鐘縮短至數秒。
4. **確認偵測結果** — 檢查稽核面板中的綠色標記，確認機器及日期欄位已被偵測。確認來源面板中僅勾選了相關的機器資料夾。若偵測失敗或資料不準確，請關閉 Smart Search 以回復為僅限 IMEI 模式。
5. **配置設定** — 選擇動作、影像類型、整理模式，並設定目標位置。
6. **搜尋** — 點擊**開始搜尋**。匹配結果將即時串流至結果表格。
7. **匯出** — 檢視結果後，點擊**匯出結果**。進度顯示於狀態列。

## 功能

| 功能 | 說明 |
|---|---|
| **多格式稽核匯入** | CSV、XLSX、XLS、TXT — 自動偵測 IMEI 欄位並略過標題列 |
| **Smart Search** | 自動偵測稽核檔案中的機器及日期欄位，進行定向資料夾查找，取代完整 NAS 掃描；自動勾選機器資料夾並填入日期範圍 |
| **平行 NAS 搜尋** | 48 個並行資料夾讀取，針對 RS3617RPxs（12 碟 RAID 5、1 Gbps）調校 |
| **平行匯出** | 8 個資料夾 x 4 個檔案 = 32 個並行複製，可充分利用 1 Gbps 頻寬 |
| **6 種整理模式** | 平鋪、依機器、依日期、機器-日期、日期-機器、依 IMEI |
| **MR PASS / FAIL 收集** | 透過切換開關從 `ModelRecogImages/` 收集模型辨識影像 |
| **僅 AI 影像** | 僅從匹配的 IMEI 資料夾匯出 `FD/` 子資料夾（AI 偵測影像） |
| **多來源管理** | 儲存並切換多個 NAS 根目錄；每個來源各自記憶其資料夾切換狀態 |
| **搜尋歷史** | 最近 5 次搜尋連同完整參數及結果摘要一併儲存 |
| **日期範圍篩選** | 將搜尋限制在特定日期區間（YYYYMMDD 資料夾名稱） |
| **掃描索引篩選** | 搜尋所有掃描或僅搜尋第一次掃描（`_1` 項目） |
| **匯出記錄** | 詳細的逐檔記錄，包含檔案大小及吞吐量；保留最近 3 份記錄 |
| **缺少 IMEI 報告** | 檢視並儲存在 NAS 上未找到的稽核 IMEI 清單 |
| **深色 / 淺色主題** | 兩種模式均採用 Liquid Glass 主題；跨工作階段保存 |
| **三語** | 英文、繁體中文及簡體中文 — 透過來源面板的語言按鈕切換 |
| **設定保存** | 所有偏好設定透過 electron-store 儲存，並於下次啟動時還原 |

## 來源配置

### 共享資料夾

輸入或瀏覽至 NAS 掛載點（例如 `Z:\`、`\\NAS_Lonestar\share`）。工具會掃描第一層子目錄並填入資料夾切換格。

- **機器資料夾**（名稱符合 `M` + 數字，如 `M8`、`M10`）預設為勾選。
- **公用資料夾**（`audits`、`crackimages`、`GRR Images`、`version_control`）預設為不勾選。
- `#recycle` 及 `$RECYCLE.BIN` 始終隱藏。

使用**全選 / 取消全選**進行批次切換。**重新整理**可重新掃描 NAS 以更新新增或移除的子資料夾。

### 多來源管理

將常用的 NAS 路徑儲存為命名來源：

1. 瀏覽至 NAS 根目錄。
2. 點擊 **[+]** 並輸入名稱（例如「NAS Lonestar」）。
3. 按下 Enter 儲存。

透過下拉選單切換來源。每個來源各自記憶其資料夾切換狀態。點擊 **[-]** 可移除來源（因為此操作無法復原，會顯示確認提示）。

首次啟動時，先前儲存的單一路徑設定會自動轉移為命名來源。

### 日期範圍

將搜尋限制在特定日期區間的資料夾：

- **起始 / 結束日期** — 僅掃描 `YYYYMMDD` 名稱落在此範圍內的資料夾。兩者皆留空則搜尋所有日期。
- 若起始日期晚於結束日期，會顯示警告（不會有任何匹配結果）。

## 稽核清單匯入

拖放或瀏覽以匯入稽核檔案。

### 建議方式：Smart Search（IMEI + 機器 + 日期）

收集影像的最快方式是在稽核檔案（CSV 或 Excel）中包含 **IMEI**、**機器**及**日期**欄位。當三個欄位皆存在時，Smart Search 會直接前往 NAS 上每個 IMEI 的確切資料夾 — 將搜尋時間從數分鐘縮短至數秒。

**建議的稽核檔案格式：**

| IMEI | 機器 | 日期 |
|---|---|---|
| 350308317018557 | M14 | 2026-05-14 |
| 350448886316469 | M21 | 2026-05-14 |
| 351101754222413 | M22 | 2026-05-14 |

欄位標題可以是任意文字 — 解析器透過資料模式識別欄位，而非依據標題名稱。欄位順序無關。額外欄位會被忽略。IMEI 欄位始終透過 15 位數字值來識別。

**機器欄位** — 可識別以下名稱格式：

| 檔案中的值 | 標準化為 |
|---|---|
| M8, M10 | M8, M10（完全匹配） |
| M08, M-8, M-08 | M8（移除前導零 / 連字號） |
| SG-M16, LAX-M08 | M16, M8（移除站點前綴） |
| Machine 8 | M8（擷取數字） |
| 08, 8 | M8（純數字） |

**日期欄位** — 可識別以下日期格式：

| 格式 | 範例 |
|---|---|
| YYYYMMDD | 20260515 |
| YYYY-MM-DD | 2026-05-15 |
| YYYY/MM/DD | 2026/05/15 |
| MM/DD/YYYY | 05/15/2026 |
| MM-DD-YYYY | 05-15-2026 |
| M/D/YYYY | 5/15/2026 |
| M/D/YY | 5/15/26 |
| M/D/YY HH:MM | 5/14/26 11:57（自動移除時間部分） |

任何尾隨的時間部分（例如 `11:57`、`11:57:00` 或 ISO `T` 分隔符）在解析前會被移除。對於模糊的日期格式（MM/DD 與 DD/MM），解析器會取樣所有資料列以判斷正確的解讀方式。完全模糊時預設使用 MM/DD（美國格式）。

**匯入後**，稽核面板以色碼標記顯示偵測品質：
- **綠色**：所有資料列解析成功
- **橘色**：80% 以上資料列解析成功（部分值無法識別）
- **紅色**：低於 80% 解析成功（欄位偵測可能不可靠）

**自動填入日期範圍**：當偵測到日期欄位時，來源面板的起始 / 結束日期欄位會自動填入稽核資料中最早及最晚的日期。這可縮小搜尋範圍，並提供日期解析正確的視覺確認。如有需要，可手動調整日期。

**自動選取機器資料夾**：當偵測到機器欄位時，來源面板會自動勾選稽核資料中引用的機器資料夾，並取消勾選其餘資料夾。例如，若稽核檔案僅包含 M14、M21 和 M22 的條目，則僅這三個資料夾會被勾選。這可避免掃描無關的機器，加快廣泛掃描的備援速度。若未偵測到機器欄位，所有機器資料夾將維持先前的切換狀態。

**部分提示**：稽核檔案不需要包含全部三個欄位。Smart Search 會根據可用的欄位進行調整：

| 可用欄位 | 搜尋行為 |
|---|---|
| IMEI + 機器 + 日期 | 直接資料夾查找 — 最快（數秒） |
| 僅 IMEI + 機器 | 廣泛掃描但限縮至每個 IMEI 提示的機器 |
| 僅 IMEI | 跨所有已選機器進行完整廣泛掃描（最慢） |

缺少值或值無法識別的 IMEI 會自動回復至下一層廣泛掃描 — 不會遺漏任何結果，這些 IMEI 只是搜尋較慢。

**MR PASS / FAIL 搜尋**同樣受益於 Smart Search。當提示可用時，MR 搜尋會直接前往 `Machine/ModelRecogImages/Date/`，而非先探索所有日期資料夾。

若偵測到的欄位不準確，請關閉 **Smart Search** 以回復為僅限 IMEI 模式。

### 替代方案：僅限 IMEI 匯入

若您的稽核資料僅包含 IMEI（無機器或日期欄位），工具會回復為廣泛 NAS 掃描 — 搜尋所有已選機器中的每個日期資料夾。此方式可行但速度明顯較慢。

| 格式 | 行為 |
|---|---|
| **CSV** | 透過取樣前 20 列自動偵測 IMEI 欄位。略過標題列。支援逗號、Tab 及分號分隔符。 |
| **XLSX / XLS** | 讀取第一個工作表。自動偵測 IMEI 欄位。略過標題列。 |
| **TXT** | 每行一個值。每行視為潛在的 IMEI。TXT 檔案無法使用 Smart Search（單一欄位）。 |

**IMEI 驗證**：必須為恰好 15 位數字。無效項目及重複項目會被計數並報告。

## 設定參考

### 動作

| 值 | 行為 |
|---|---|
| **複製**（預設） | 將匹配的資料夾複製至目標位置。來源資料不受影響。 |
| **移動** | 傳輸資料夾並在複製後**刪除來源**。啟用前會顯示確認對話框。下次啟動時，動作始終重設為複製以確保安全。 |

> 移動模式**不適用於** MR 影像匯出 — MR 影像始終以複製方式處理，以保護共享的 `ModelRecogImages/` 目錄。

### 影像類型

| 值 | 行為 |
|---|---|
| **全部**（預設） | 匯出匹配資料夾中的所有影像檔案。 |
| **BMP** | 僅匯出 `.bmp` 檔案。 |
| **JPEG** | 僅匯出 `.jpg` / `.jpeg` 檔案。 |

### 整理

控制匯出資料夾在目標位置的排列方式：

| 模式 | 結構 | 範例 |
|---|---|---|
| **平鋪** | 全部放在同一資料夾 | `dest/350002267153742_192/` |
| **依機器** | 依來源機器分組 | `dest/M8/350002267153742_192/` |
| **依日期** | 依掃描日期分組 | `dest/20260515/350002267153742_192/` |
| **機器 > 日期** | 兩層：先機器再日期 | `dest/M8/20260515/350002267153742_192/` |
| **日期 > 機器** | 兩層：先日期再機器 | `dest/20260515/M8/350002267153742_192/` |
| **依 IMEI** | 將同一裝置的所有實例分組 | `dest/350002267153742/M8_20260515_192/` |

**MR 影像匯出**時，每個 IMEI 會有自己的資料夾，內含匹配的 `.png` 檔案：

| 模式 | MR 範例 |
|---|---|
| 平鋪 | `dest/350002267153742/M8_20260515_Samsung-Galaxy_S24.png` |
| 機器 > 日期 | `dest/M8/20260515/350002267153742/Samsung-Galaxy_S24.png` |
| 依 IMEI | `dest/350002267153742/M8_20260515_Samsung-Galaxy_S24.png` |

> **提示：** 整理模式獨立於搜尋 — 您可以在搜尋完成後、匯出前更改模式。匯出將使用您點擊**匯出結果**時所選的模式。

### 重複

| 值 | 行為 |
|---|---|
| **略過**（預設） | 若目標資料夾 / 檔案已存在，不進行變更。 |
| **覆蓋** | 以新的來源資料取代現有的目標資料夾 / 檔案。 |

### 掃描索引

| 值 | 行為 |
|---|---|
| **全部**（預設） | 包含每個掃描索引（`_1`、`_2`、`_3` 等）。 |
| **僅第一次掃描** | 僅 `_1` 項目（該裝置在該機器 / 日期的第一次掃描）。 |

### MR PASS

收集**模型辨識 PASS** 影像 — AI 正確識別的裝置。

- 搜尋 `Machine/ModelRecogImages/{date}/{Brand-Model}/` 資料夾。
- 匹配檔名包含稽核清單中 15 位 IMEI 的 `.png` 檔案（格式：`SG-{machine}-{code}-{IMEI}-{brand}-{model}.png`）。
- 結果表格中以綠色顯示 Brand-Model 資料夾名稱（例如 `Apple-iPhone8`）。
- 啟用時會**停用標準影像收集**。

### MR FAIL

收集**模型辨識 FAIL** 影像 — AI 誤判的裝置。

- 搜尋 `Machine/ModelRecogImages/{date}/Error-Error/` 資料夾。
- 與 MR PASS 相同的匹配邏輯。
- 結果表格中以紅色顯示 `Error-Error`。

> MR PASS 及 MR FAIL 可同時啟用。搜尋會合併 Brand-Model 及 Error-Error 子資料夾的結果。

### 僅 AI 影像

啟用時，僅從每個匹配的 IMEI 資料夾匯出 `FD/` 子資料夾內容（AI 偵測影像）。資料夾根目錄的標準掃描影像會被排除。

停用時，會匯出完整的 IMEI 資料夾，包含 `FD/` 作為子資料夾。

### 目標位置

匯出影像寫入的本機資料夾。瀏覽或輸入路徑。跨工作階段保存。

## 結果

搜尋完成後，結果面板顯示：

- **摘要列** — 找到的 IMEI 總數 vs. 稽核清單中的總數、經過時間。
- **狀態圓點** — 綠色（完整）、橘色（不完整：檔案數量明顯低於中位數）、紅色（缺少：NAS 上未找到）。
- **可排序表格** — 點擊任何欄位標題（IMEI、機器、日期、索引、檔案）進行升冪 / 降冪排序。
- **檔案明細** — 每個匹配項目顯示總檔案數，並以 `(Xb Yj)` 格式顯示 BMP 及 JPEG 數量。
- **檢視缺少的 IMEI** — 切換表格以顯示未匹配的稽核項目。
- **儲存缺少的 IMEI** — 將缺少清單匯出為 `.txt` 或 `.csv` 檔案。

### 不完整偵測

當匹配項目的檔案數量低於所有匹配項目檔案數量中位數的 50% 時，會被標記為不完整（橘色圓點）。這可以發現存在但影像數量異常少的 IMEI 資料夾。

## 匯出記錄

每次匯出都會在應用程式的使用者資料目錄中產生詳細的記錄檔：

```
%APPDATA%/Image Collection v2/logs/export-YYYYMMDD-HHmmss.log
```

每份記錄包含：
- **標題** — 時間戳記、目標位置、所有設定、並行設定。
- **逐資料夾項目** — 來源路徑、目標路徑、每個已複製的檔案及其大小、經過時間。
- **摘要** — 已複製的總檔案數及位元組數、吞吐量（MB/s）、略過 / 失敗數量。

工具保留**最近 3 份**記錄檔，並自動輪替較舊的檔案。

匯出後點擊狀態列中的**檢視記錄**以開啟記錄資料夾。

## 搜尋歷史

最近 5 次已完成的搜尋會被儲存，可透過操作列中的**歷史**按鈕存取。每個項目顯示：

- 搜尋的日期和時間
- 來源名稱及 NAS 路徑
- 稽核檔案名稱及 IMEI 數量
- 匹配數量及缺少數量
- 搜尋經過時間
- 日期範圍（若有篩選）
- MR 標記（若 MR 切換為啟用）

歷史紀錄跨工作階段保存。

## NAS 目錄結構

工具預期 NAS 上具有以下階層結構：

```
NAS_ROOT/                              （例如 Z:\）
  M8/                                  機器資料夾（第 1 層）
    20260515/                           日期資料夾（第 2 層，YYYYMMDD）
      350002267153742_192/              IMEI_索引資料夾（第 3 層）
        image1.bmp                      影像檔案（第 4 層）
        image2.jpg
        FD/                             AI 偵測影像子資料夾
          fd_image1.bmp
      350024510270586_85/
    20260514/
    ModelRecogImages/                   MR 影像（MR 切換為 ON 時搜尋）
      20260515/
        Apple-iPhone8/                  Brand-Model 子資料夾（MR PASS）
          SG-M008-075545-358627090247469-Apple-iPhone8.png
        Error-Error/                    錯誤子資料夾（MR FAIL）
          SG-M008-074837-359814715825890-Apple-iPhoneXR.png
    Bin/                                搜尋時略過
  M10/
  ...
  #recycle/                             始終隱藏
  audits/                               公用資料夾（預設不勾選）
```

**命名慣例：**
- IMEI：始終為 15 位數字。
- 掃描索引：底線後的整數（`_1` = 第一次掃描、`_2` = 重新掃描，依此類推）。
- 日期資料夾：始終為 8 位數字，`YYYYMMDD` 格式。
- MR 影像檔名：`SG-{machine}-{code}-{IMEI}-{brand}-{model}.png`（IMEI 為第 4 個連字號分隔的區段）。

## 從原始碼建置

### 前置需求

- [Node.js](https://nodejs.org/) v18 或更新版本
- npm（隨 Node.js 附帶）

### 開發

```bash
git clone <repo-url>
cd Image-Collection-v2
npm install
npm run dev          # 啟動並啟用熱重載
```

### 正式建置

```bash
npm run build        # 編譯 main + preload + renderer
npm run pack         # 建置 + 建立未封裝目錄（快速，用於測試）
npm run dist         # 建置 + 建立 NSIS 安裝程式（.exe）
```

安裝程式輸出至 `release/Image Collection v2 Setup X.X.X.exe`。

### 其他指令

| 指令 | 說明 |
|---|---|
| `npm run typecheck` | 對 main 及 renderer 執行 TypeScript 類型檢查 |
| `npm run lint` | 使用 ESLint 檢查原始碼 |
| `npm run format` | 使用 Prettier 格式化原始碼 |

## 安裝

執行 NSIS 安裝程式。它將：

- 安裝至 `C:\Program Files\Image Collection v2\`
- 建立**桌面捷徑**及**開始功能表捷徑**
- 在**新增或移除程式**中註冊，以便完整解除安裝
- 可選擇在安裝完成後啟動應用程式（完成頁面的核取方塊）

安裝後不需要管理員權限。應用程式將設定及記錄儲存於 `%APPDATA%/Image Collection v2/`。

### 解除安裝

使用**新增或移除程式**（設定 > 應用程式 > 已安裝的應用程式）。解除安裝程式會移除所有檔案、捷徑、登錄機碼及應用程式資料 — 零殘留。

## 系統需求

- **作業系統**：Windows 10 或更新版本（64 位元）
- **NAS 連線**：NAS 共享必須已掛載或可透過 UNC 路徑存取（例如 `\\NAS_Lonestar\share`）
- **磁碟空間**：應用程式約需 200 MB；目標磁碟需要足夠的空間存放匯出影像
- **無執行階段相依性**：Node.js 已與 Electron 應用程式打包在一起

## 已知限制

- **AutoMode** 及 **Rescan Image Collection** 頁籤已延後，等待工程團隊提供實作規格。
- 預設的應用程式圖示（`resources/icon.ico`）是一個產生的藍綠色方塊。在最終發布前請替換為正式圖示。
- 日期範圍篩選依據資料夾日期名稱（`YYYYMMDD`）運作。不支援日內時間篩選 — NAS 資料夾結構不包含時間編碼。
- 帶有色碼的 CSV/Excel 匯出摘要報告已規劃但尚未實作。
